Funcionalidades

7 módulos listos para demostrar

📒

Gestión de Ledgers

CRUD completo con ciclo de vida de estados, selector de país global y registro de auditoría automático.

  • Crear / editar / activar / desactivar
  • Estados: Borrador → Activo → Inactivo
  • Selector de país 🇨🇴 🇵🇪 en header global
  • Versioning de configuraciones
⚖️

Configuración Contable

AccountingEntryConfig con autocompletado de cuentas PUC, validación de partida doble y preview de asiento.

  • Autocompletado de cuenta auxiliar (código o nombre)
  • Naturaleza Débito / Crédito
  • Validación ∑Débitos = ∑Créditos
  • Preview asiento con datos ejemplo
📊

Dashboard de Monitoreo

Métricas en tiempo real con auto-refresh configurable y alertas activas.

  • KPIs: latencia P50/P95/P99, tasa errores
  • Gráfico transacciones por hora (24h)
  • Alertas de latencia y tasa de error
  • Auto-refresh 15s / 30s / 60s
🧾

Log de Transacciones

Búsqueda multi-criterio con detalle completo de payload y asientos generados.

  • Filtros: ID, cliente, estado, ledger, fecha
  • Detalle: payload, asientos, latencia, errores
  • Exportar hasta 10.000 registros a CSV
  • Link directo shareable por transacción
🛡️

Auditoría de Cambios

Log inmutable append-only con diff visual antes/después para compliance SFC.

  • Registro automático de todos los cambios
  • Diff visual antes / después
  • Filtros por usuario, acción, fecha, ledger
  • Exportar para auditorías externas
🗂️

Plan de Cuentas (PUC)

Catálogo de cuentas contables colombiano con CRUD y autocomplete integrado en la configuración contable.

  • 25 cuentas PUC Colombia precargadas
  • Tipos: Activo / Pasivo / Ingreso / Egreso
  • Búsqueda por código o nombre
  • Exportar / Importar CSV
🔌

Configuración ERP

Preparación para integración futura con ERP externo (Fase 2). Filtrado por país activo.

  • Formatos: CSV / JSON / XML
  • Frecuencia: Diaria / Semanal / Mensual
  • Mapeo campos Ledger → ERP
  • Preview de archivo exportado

Casos de uso

Flujos principales implementados

⚡ Crear y activar ledger (Happy Path)

1
Seleccionar país en header (🇨🇴 Colombia o 🇵🇪 Perú)
2
"Crear Ledger" → Nombre, Compañía, Moneda, Frecuencia
3
Guardar como Borrador → redirige a detalle del ledger
4
Tab "Configuración Contable" → agregar transaction_type
5
Buscar cuenta auxiliar → mapear campos → validar partida doble

🔍 Investigar transacción con error

1
Panel de Alertas → "Tasa de errores: 2.5%"
2
Click en alerta → Log filtrado por ledger + status=failed
3
Abrir detalle → ver payload + mensaje de error
4
Link rápido a configuración del ledger → editar mapeo
5
Guardar → toast "✓ Configuración actualizada"
